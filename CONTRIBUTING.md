# Como contribuir a PlayStop Frontend

## Flujo de trabajo

1. Crear una rama: git checkout -b feature/nombre-funcionalidad
2. Desarrollar los cambios necesarios
3. Ejecutar el linter: npm run lint
4. Formatear el codigo: npm run format
5. Crear commits descriptivos en espanol
6. Abrir un Pull Request hacia main

## Convenciones de commits

Usar mensajes descriptivos en espanol:
- Agregar [funcionalidad]
- Corregir [bug]
- Mejorar [componente]
- Refactorizar [modulo]
- Eliminar [codigo obsoleto]

## Estandares de codigo

- Componentes funcionales con hooks de React
- Nombres de componentes en PascalCase
- Nombres de funciones en camelCase
- Estilos con Tailwind CSS y clases utilitarias
